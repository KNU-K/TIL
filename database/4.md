# 데이터베이스 시스템(DBS)

데이터 베이스를 유지 및 관리하고 필요한 시점에 데이터를 입력, 조작, 그리고 변경하여 어떤 목적 달성을 위해 이용하는 시스템

- DBS를 구성하는 요소
  > 데이터베이스, DBMS, 데이터베이스 언어, USER, DBA, HW etc..

## Schema in database

데이터베이스 구조(structure)와 데이터 제약 조건(constraints)에 대한 논리적인 정의를 기술하는 것

> 스키마에는 개체 속성 관계에 대한 유지 조건을 포함

1. 외부 스키마 (external schema)

   각각의 사용자나 응용 프로그래머가 접근하는 DB의 logical sub-schema 를 정의

2. 개념 스키마 (conceptual schema)

   조직 전체의 총괄적 입장에서 본 DB 기술

3. 내부 스키마 (internal schema)

   개념 스키마에 대한 내부 물리적인 저장 구조를 기술

## 데이터베이스 언어

- DDL (데이터 정의어)

  데이터베이스 스키마를 컴퓨터가 이해할 수 있도록 기술하는데 사용

- DML (데이터 조작어)

  데이타 처리(CRUD)을 지원

  - 절차적 데이터 조작어

    > 범용 프로그램 언어 속에 삽입(embedded)된 형태로 사용

  - 비절차적 데이터 조작어

    > query language처럼 일반 사용자가 대화식으로 사용 ( 고급 명령어 )

- DCL (데이터 제어어)
  데이터 제어(보안, 무결성 회복 병행수행)를 정의하고 기술하는 언어

> DDL, DML, DCL로 구별핮만, 현실적으로 이들을 위한 별도의 언어로 존재하지않고, 하나의 표준화 안에서 통합되어 사용

# 사용자 (User)

- 일반 사용자

  비절차적 DML 언어를 통해 DB에 접근

- 응용 프로그래머

  범용 프로그래밍 언어와 DBMS의 절차적 DML 언어를 사용하여 DB를 접근하고 개발

- 데이터베이스 관리자

  DDL과 DCL 을 통해 DB를 정의하고 제어

  > DBS의 관리 운영에 대한 모든 책임을 가지는 사람

  - DBA의 업무

    - DB 설계와 운영

    - 행정적인 업무

    - 시스템 모니터링 및 각종 통계정보 분석
